Ordered Weighted Averaging

The diving example mentioned in the topic 'Trimmed & Winsorized Mean' is an example of ordered weighted averaging, where weights are applied after the scores are ordered.

The ordered weighted averaging operator (OWA) can be written as follows:

OWAw(x)=∑i=1nwix(i)=w1x(1)+w2x(2)+⋯+wnx(n){"version":"1.1","math":"\\ OWA_\mathbf{w}(\mathbf{x}) = \sum_{i=1}^n w_i x_{(i)} = w_1x_{(1)} + w_2x_{(2)} + \dots + w_nx_{(n)} \\"}

Where x(1),x(2),…,x(n){"version":"1.1","math":"\(x_{(1)}, x_{(2)}, \dots, x_{(n)}\)"} is obtained by rearranging x1,x2,…,xn{"version":"1.1","math":"\(x_1, x_2, \dots, x_n\)"} into non-decreasing order.  The rearranged vector is denoted in this course as x↗{"version":"1.1","math":"\(x_{\nearrow}\)"} (although this is not universal notation).

Example: Ordering influence

Suppose x=(0.3,0.6,0.8,0.2){"version":"1.1","math":"\(\mathbf{x} = (0.3, 0.6, 0.8, 0.2)\)"} and w=(0.1,0.3,0.4,0.2){"version":"1.1","math":"\(\mathbf{w} = (0.1, 0.3, 0.4, 0.2)\)"}.  Ordering x{"version":"1.1","math":"\(x\)"} gives x↗=(0.2,0.3,0.6,0.8){"version":"1.1","math":"\(x_{\nearrow} = (0.2, 0.3, 0.6, 0.8)\)"} and thus OWAw(x)=(0.1×0.2)+(0.3×0.3)+(0.4×0.6)+(0.2×0.8)=0.51{"version":"1.1","math":"\\ OWA_\mathbf{w}(\mathbf{x}) = (0.1 \times 0.2) + (0.3 \times 0.3) + (0.4 \times 0.6) + (0.2 \times 0.8) = 0.51 \\"}

The OWA is useful because it allows us to define functions that graduate between the minimum and maximum function, favouring higher or lower inputs depending on our preference.

For example:  w=(0,0,0.1,0.2,0.3,0.4){"version":"1.1","math":"\(\mathbf w = (0, 0, 0.1, 0.2, 0.3, 0.4)\)"} 

takes the highest 4 inputs into account and gives the highest one the most influence. We might have a function like this kind of aggregation with student tests if we want to take their best test scores into account but also reward consistently high performance.

Properties

Despite it using a weighting vector, the OWA satisfies the following properties

  • symmetry - order of numbers doesn’t affect the output.
  • translation invariant - adding a constant to every value will increase the output by that same constant.
  • homogenous - multiplying all values by the same constant will increase the output by the same factor.
  • monotone - an increase in the input will result in an increase in the output.
  • idempotent - if all values are the same, then the output will be the same value.
