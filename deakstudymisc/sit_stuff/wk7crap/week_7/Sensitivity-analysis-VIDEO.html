Sensitivity Analysis

How much flexibility do we have with our current solution?

Often times clients will want to know how much leeway and/or flexibility they have with their resources. Sensitivity analysis is the term given to this area and the types of analyses used is dependent on context (i.e. not restricted to linear programming). Let us approach this using the LP from 1.06 Toy Company which we repeat here below.

The video above demonstrates a visual and interactive way of approaching it, which you can do yourself in Desmos as well. This should hopefully give you a few ideas and insights to help reinforce the analysis below.

The question we want to answer in this analysis below is:

How would changes in the problem’s objective function coefficients affect the optimal solution? That is, in the objective function 3x1+2x2{"version":"1.1","math":"\(3x_1 + 2x_2\)"}, what happens if we changed the 3 and/or the 2? In terms of the original problem, this is basically asking ‘What is the range of profit I can make off a soldier or a train?’

We’ll address this problem with respect to the soldier.

The way to approach this analytically is to rewrite all the constraints in ‘slope-intercept’ form, which is the form y=mx+c{"version":"1.1","math":"\(y = mx+c\)"} or a similar variant, where m{"version":"1.1","math":"\(m\)"} is the slope and c{"version":"1.1","math":"\(c\)"} is the y{"version":"1.1","math":"\(y\)"}-intercept. The goal is to identify the slope of each of the constraints and the objective function at the optimal solution.

160=3x+2y⟹y=−32x+1602x+y≤100⟹y≤−2x+100x+y≤80⟹y≤−x+80{"version":"1.1","math":"% <![CDATA[ \\ \begin{align} 160 = 3x+2y & \implies y = -\frac{3}{2}x + 160 \\ 2x+y \leq 100 & \implies y \leq -2x + 100 \\ x+y \leq 80 & \implies y \leq -x + 80 \end{align} \\ %]>"}

Graphical Method Slopes Image 

If we pivot at (40,20){"version":"1.1","math":"\((40, 20)\)"} in a clockwise direction, making the slope of the objective function approach -2, we approach the slope of the finishing constraint. Beyond this point, the optimal solution changes from (40,20){"version":"1.1","math":"\((40,20)\)"} to (50,0){"version":"1.1","math":"\((50,0)\)"}.

Moving in an anticlockwise direction, the slope of the objective function does not approach any constraint in particular as it moves towards 0. However our optimal solution still won’t change.

Therefore, to keep the current solution optimal, the slope of the objective function would have to be between -2 and 0 for the current optimal solution to remain optimal. We can technically move beyond 0

The way to write this algebraically is to start with the objective function, but to replace the coefficient of interest with a dummy variable (we use c{"version":"1.1","math":"\(c\)"} here).

z=cx+2y⟹y=−c2x+z{"version":"1.1","math":"% <![CDATA[ \\ \begin{align} z & = cx + 2y \\ \implies y & = -\frac{c}{2}x + z \\ \end{align} %]>"}

Since we’ve established the slope to be between -2 and 0, we can write:

−2≤−c2≤0{"version":"1.1","math":"\\ -2 \leq -\frac{c}{2} \leq 0 \\"}

Multiplying all sides by -2 (note that multiplying by a negative number flips the signs) gives

0≤c≤4{"version":"1.1","math":"\\ 0 \leq c \leq 4 \\ "}

Therefore the current solution would remain optimal as long as the first coefficient (i.e. profit per soldier) is between 0 and 4. If we go beyond 4, then we’d need to adjust production from making 40 soldiers and 20 trains, to making 50 soldiers and 0 trains instead to maximise our profit.
