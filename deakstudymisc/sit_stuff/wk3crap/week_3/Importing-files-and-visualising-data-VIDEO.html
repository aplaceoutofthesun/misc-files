Importing files and visualising data

Any data we import into RStudio will typically be from some external source, for example excel files.

Here we’ll look at some of the basic methods for importing data from files and how to visualise them.

The two datasets used in the video can be downloaded via the links below. We definitely recommend doing so and following along with the video.

  • volley.txt
  • ebola.csv

Importing Data - Extra Features

The first thing we’ll do here is refer to the documentation which refers to both read.table and read.csv

We’ll explain some of the more common arguments that beginners would be interested in using, and suggest commands you can enter into the console to verify those changes.

  • header - this only takes a TRUE or FALSE value, and we want it to be TRUE when our dataset’s first row contains what looks like headers.
    We saw this in ebola.csv, but not volley.txt.

Notice in the documentation how for read.table this is set to FALSE by default, and for read.csv this is set to ‘TRUE’ by default.

  • Try importing volley.txt with the header set to ‘TRUE’:
    read.table("volley.txt", header = TRUE)
    • view the imported data. What happened?
  • Try importing ebola.csv with the header set to ‘FALSE’:
    read.csv("ebola.csv", header = FALSE).
    • What happened?
  • sep - stands for ‘separator’ and refers to the character(s) used to separate values within the data.
    • In a CSV (comma separated values) file, this is set to the comma character by default. It’s rare to see separators in data that aren’t spaces, tabs or commas, so we need not worry about this too much.
  • col.names - allows us to specify the column names when we’re importing data.
    • The way we do this is by using an array of strings e.g. c("column1", "column2", "column3"). By default the column names use V1, V2, V3, etc.
  • Try importing volley.txt and specify column names for what your best guess the values represent:
    read.table("volley.txt", col.names = c("column1", "column2", "column3", "column4"))

There are a few more options that we’ll address in the next R Tutorial!

Visualising Data - Extra Features

Once again, referring to the documentation for the plot function shows us what additional things we can do.

The most relevant arguments that all take a single string as their argument are:

  • main
  • sub
  • xlab (x-axis label)
  • ylab (y-axis label)

Try redoing your scatterplot with data from volley.txt and ensure that it has a title and all axes are named appropriately.
