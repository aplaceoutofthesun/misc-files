Arithmetic Mean

The arithmetic mean is commonly referred to as the ‘average’, ‘statistical mean’ or just the ‘mean’ and is at the heart of the majority of aggregation processes.

The arithmetic mean (AM) of a set of values x1,x2,…,xn{"version":"1.1","math":"\(x_1, x_2, \dots, x_n\)"} is as follows:

AM(x1,x2,…,xn)=x1+x2+⋯+xnn=1n∑i=1nxi.{"version":"1.1","math":"AM(x_1, x_2, \dots, x_n) = \frac{x_1 + x_2 + \cdots + x_n}{n} = \frac{1}{n} \sum_{i=1}^n x_i. "}

Using arithmetic mean

The above is applied in the following example:

Let x=(3,2,9,8){"version":"1.1","math":"\(\mathbf{x} = (3, 2, 9, 8)\)"}.

1. What is the value of n{"version":"1.1","math":"\(n\)"}?

The value of n{"version":"1.1","math":"\(n\)"} is 4{"version":"1.1","math":"\(4\)"}, as there are 4 numbers in x{"version":"1.1","math":"\(x\)"}.

2. Calculate AM(x){"version":"1.1","math":"\(AM(\textbf{x})\)"}.

AM(x)=3+2+9+84=224=5.5{"version":"1.1","math":"AM(\mathbf{x}) = \frac{3+2+9+8}{4} = \frac{22}{4} = 5.5"}

3. What will be the output if x=(9,8,2,3){"version":"1.1","math":"\(\mathbf{x} = (9, 8, 2, 3)\)"}?

AM(9,8,2,3)=9+8+2+34=5.5{"version":"1.1","math":"AM(9,8,2,3) = \frac{9+8+2+3}{4} = 5.5"}

The arithmetic mean has a few properties which allow us to compare it to other aggregation functions:

  • symmetry - the order of the numbers doesn’t affect the output, as demonstrated in the above example.
  • translation invariant - if we add a constant to every value and take the average, then the output will change by that same constant. e.g. AM(4,3,8)=5{"version":"1.1","math":"\(AM(4,3,8) = 5 \)"} and AM(7,6,11)=8{"version":"1.1","math":"\(AM(7,6,11) = 8\)"}, showing that adding 3{"version":"1.1","math":"\(3\)"}  to all values also increases the arithmetic mean by 3{"version":"1.1","math":"\(3\)"}.
  • homogeneous - if we multiply all values by a constant, the output will also change by the same factor e.g. AM(3,7,8)=6{"version":"1.1","math":"\(AM(3,7,8) = 6\)"} and AM(15,35,40)=30{"version":"1.1","math":"\(AM(15,35,40) = 30\)"}, showing that multiplying all values by 5{"version":"1.1","math":"\(5\)"} also multiplies the arithmetic mean by 5{"version":"1.1","math":"\(5\)"}.
  • monotone increasing - increasing any one of the inputs always increases the output e.g. AM(40,57,29)<AM(40,57,30){"version":"1.1","math":"\(AM(40, 57, 29) < AM(40, 57, 30)\)"}, where increasing the value from 29{"version":"1.1","math":"\(29\)"} to 30{"version":"1.1","math":"\(30\)"} will result in a higher result.
  • idempotent - if all values are the same, then the output will also be the same value e.g. AM(5,5,5,5)=5{"version":"1.1","math":"\(AM(5, 5, 5, 5) = 5\)"}.

Activity

Determine a set of values where the arithmetic mean does not appear to accurately reflect the ‘centre’ of the values. Are there any other measures of central tendency which might give a better impression of the ‘centre’ of those values?

In R, the function mean() allows you to calculate the mean of a given vector. Typing mean(c(9,8,2,3)) in the console should give you 5.5{"version":"1.1","math":"\(5.5\)"} ,verifying the result calculated in the example.

If you are given the mean of n{"version":"1.1","math":"\(n\)"} values, how many values would you need to know to determine all of the remaining values? For further reading, look up ‘degrees of freedom’.
