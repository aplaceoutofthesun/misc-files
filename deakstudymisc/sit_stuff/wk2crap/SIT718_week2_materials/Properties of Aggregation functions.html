Properties of Aggregation functions

There are common properties between all the functions this week that allow us to classify them as aggregation functions.

More formally, for an input vector x=(x1,x2,…,xn){"version":"1.1","math":"\(\mathbf{x} = (x_1, x_2, \dots, x_n)\)"}, a function A(x){"version":"1.1","math":"\(A(\mathbf{x})\)"} is an aggregation function if it:

  • is monotone increasing i.e. an increase in any of the inputs does not result in a decrease in the output.

In other words, if we increase any of the inputs of x{"version":"1.1","math":"\(\mathbf{x}\)"}, then A(x){"version":"1.1","math":"\(A(\mathbf{x})\)"} will either be larger or remain the same, but never smaller. This can be expressed mathematically as

if x≤y then A(x)≤A(y){"version":"1.1","math":"\\ \text{if } \mathbf{x} \leq \mathbf{y} \text{ then } A(\mathbf{x}) \leq A(\mathbf{y}) \\"}

  • satisfies the boundary conditions. If a{"version":"1.1","math":"\(a\)"} and b{"version":"1.1","math":"\(b\)"} represent the minimum and maximum values respectively, then

 A(a,a,…,a)=a and A(b,b,…,b){"version":"1.1","math":"\\ A(a,a, \dots, a) = a \text{ and } A(b,b, \dots, b) \\"}

For example, if all our values are in the unit interval [0,1], then we can expect A(0,0,…,0)=0{"version":"1.1","math":"\(A(0,0, \dots, 0) = 0\)"} and A(1,1,…,1)=1{"version":"1.1","math":"\(A(1,1,\dots, 1) = 1\)"}. Similarly, if our data range is from 1 to 1000 then we would require A(0,0,…,0)=0{"version":"1.1","math":"\(A(0,0, \dots, 0) = 0\)"} and A(1000,1000,…,1000)=1000{"version":"1.1","math":"\(A(1000,1000,\dots, 1000) = 1000\)"}

A function like A(x,y)=xy{"version":"1.1","math":"\(A(x,y) = xy\)"} is an aggregation function over the interval [0,1], but not over the interval [1,10] since it does not satisfy the boundary conditions: A(10,10)=10×10=100≠10{"version":"1.1","math":"\(A(10, 10) = 10 \times 10 = 100 \neq 10\)"}.

A subclass of aggregation functions are averaging functions whose output are bounded between the minimum and maximum of its inputs i.e.

min(x)≤A(x)≤max(x).{"version":"1.1","math":"\\ \min(\mathbf{x}) \leq A(\mathbf{x}) \leq \max(\mathbf{x}). \\"}

While this is not universal terminology, aggregation functions falling into this category have the convenience of their results being interpreted on the same scale and units as the inputs.

Activity

Verify that all the aggregation functions this week satisfy the monotone increasing property by generating a random array and calculating its output. Increase one of the values of your array and observe whether your output increases or not. Some hints:

  • The function runif(n, min = a, max = b) generates an array of n{"version":"1.1","math":"\(n\)"} random numbers between a{"version":"1.1","math":"\(a\)"} and b{"version":"1.1","math":"\(b\)"}.  Type ?runif in the console to view the documentation for this function.
  • We can replace a single element within an array by assigning it a new value. Let’s say we have an array arr = c(2, 5, 8, 9), then we can replace the second value by typing arr[2] = 6.

Which of the aggregation functions this week are averaging aggregation functions and how can we verify this?

Referring back to the properties mentioned in the topic 'Arithmetic Mean'.

  • symmetry - the order of the numbers doesn’t affect the output.
  • translation invariant - if we add a constant to every value and take the average, then the output will change by that same constant.
  • homogeneous - if we multiply all values by a constant, the output will also change by the same factor.
  • idempotent - if all values are the same, then the output will also be the same value. Which of these properties also apply to the other aggregation functions?
