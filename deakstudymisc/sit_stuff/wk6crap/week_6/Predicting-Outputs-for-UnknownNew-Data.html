hotel977 = c(59,85,66,55,78,91,87,94,73)

The general process is:

1. Transform our inputs using the same transformations used in our transformation step.

Since the only transformation we did in our transformation step was divide all inputs by 100, we do the same thing here.

hotel2 = hotel2/100
hotel826 = hotel826/100
hotel977 = hotel977/100

2. Using our model(s) of choice, combine our input with the weights acquired from the previous step.

For the WAM:

WAMweights = c(0.0194056285221035,0.0390876342948007,0.102891478464021,0.240427752211895,
0.281991744682169,0.0470443873526975,0,0.110916927376011,0.158234447096302)
QAM(hotel2, WAMweights) # 0.7032886
QAM(hotel826, WAMweights) # 0.7663735
QAM(hotel977, WAMweights) # 0.7273545

For the Choquet Integral, using the truncated version and accounting for the shifted inputs:

Choquetweights = c(0.179487179487191,0.214285714285701,0.400000000000108,
0.117216117216137,0.179487179487191,0.954545454545463,
0.954545454545558,0.0357142857143169,0.892857142856606,
0.214285714285695,1,0.142857142857174,
1,1,1)
choquet(hotel2[c(5,4,9,8)], Choquetweights) # 0.6739286
choquet(hotel826[c(5,4,9,8)], Choquetweights) # 0.8691558
choquet(hotel977[c(5,4,9,8)], Choquetweights) # 0.7464286

3. Transform our results back using the inverses of the appropriate transformations, and check for consistency as needed.

Since all of Kei’s prior ratings were divided by 100 when determining weights, we need only multiply our results by 100 to obtain Kei’s predicted ratings. At a glance we can see the relative rankings are consistent with Hotel 826 having the highest rating over Hotel 977, then Hotel 2. If this wasn’t the case then it’s an indicator that something may have gone wrong somewhere.

The accuracy of the actual numbers itself can still be subject to question, and we can run another gamut of tests/checks (same as the ones we did while exploring our data) to ascertain our hypotheses.
